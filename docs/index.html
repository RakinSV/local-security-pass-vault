CSV Export

Export all logins to Chrome/Firefox-compatible CSV. Import back from Chrome, Firefox, Bitwarden, and 1Password.

Browser Extension

Chrome, Edge, Firefox auto-fill via local native messaging. Zero network requests — all data stays on your machine.

OS Quick-Unlock

After first login, use Windows DPAPI / macOS Keychain / libsecret for quick vault unlock. Removable from Settings.

BIP-39 Backup

24-word seed phrase for encrypted .vbk backups. Shown once, never stored on disk. Argon2id 4 GB KDF — GPU-proof.

Multi-Vault

Separate encrypted databases for work, personal, and family. Each vault has its own master password and keychain entry.

⏱

Auto-Lock

Configurable idle timer (1 min to 1 hour). Lock on minimize. Vault key zeroed from memory immediately on lock.

Military-grade cryptography, properly implemented
Built on libsodium — the same library used in WireGuard, Signal Protocol implementations, and Tor.
MASTER PASSWORD + 32-byte random Salt │ ▼ ARGON2ID KDF (m=256 MB · t=4 iterations · p=4 threads) │ ├──► db_key (32 B) → SQLCipher AES-256 (database-level encryption) ├──► enc_key (32 B) → decrypts the Vault Key └──► search_key (32 B) → HMAC-SHA256 search index Vault Key (random 32 B, created once at vault init) │ ▼ XChaCha20-Poly1305 (192-bit nonce, unique per save) │ ▼ Encrypted records in SQLCipher DB RTX 4090 brute-force: ~2-4 seconds per attempt · practical attacks impossible
🛡

Envelope Encryption

Master password derives keys, not encrypts data directly. Changing your password re-encrypts only the Vault Key — records untouched.

⚡

Unique Nonce Per Save

Every record gets a fresh 192-bit random nonce at each save. Nonce reuse — the most common AEAD implementation bug — is architecturally impossible.

🔒

Memory Zeroing

sodium_memzero() + mlock() on every key. Keys can't be swapped to disk. On lock, the vault key is immediately zeroed from RAM.

🐛

Ransomware Detection

A honeypot file sits next to vault.db. Any unauthorized modification triggers an immediate vault lock and user alert.
