# Модель угроз VaultPass

**Версия:** 1.0  
**Дата:** 2026-06  
**Следующий пересмотр:** 2027-06

## Что защищаем

- Мастер-пароль пользователя
- Vault Key (ключ шифрования хранилища)
- Расшифрованные пароли записей в памяти
- Файл vault.db на диске
- Канал IPC между расширением и десктопом

## Допущения безопасности (Trusted Computing Base)

Мы ДОВЕРЯЕМ:
- Ядру ОС (если ядро скомпрометировано — мы не можем защититься)
- Производителю процессора (Spectre/Meltdown в модели — принять риск)
- libsodium (аудированная библиотека, статически вшита)
- Пользователю (он не враг себе сам)

Мы НЕ ДОВЕРЯЕМ:
- Другим процессам на машине
- Файловой системе (symlink атаки)
- Сети (расширение не делает сетевых запросов)
- Браузерным страницам (content script изолирован)

## Матрица угроз

| Угроза | Вероятность | Ущерб | Контрмеры |
|--------|------------|-------|-----------|
| Брутфорс vault.db | Средняя | Критический | Argon2id m=256MB |
| Memory dump малварью | Высокая | Критический | mlock + memzero + SetProcessMitigationPolicy |
| Keylogger мастер-пароля | Высокая | Критический | Виртуальная клавиатура (частично) |
| DLL hijacking | Средняя | Критический | Статическая линковка libsodium |
| Ransomware шифрует vault | Средняя | Высокий | readonly + honeypot + VSS |
| Clipboard перехват | Высокая | Высокий | TTL 30 сек + EXCLUDE_FROM_SYNC |
| Screen capture | Средняя | Высокий | WDA_EXCLUDEFROMCAPTURE |
| Symlink атака | Низкая | Высокий | O_NOFOLLOW + lstat() |
| Supply chain (зависимости) | Низкая | Критический | cargo audit + pinned versions |
| XSS в popup расширения | Низкая | Критический | CSP + isolated world |
| Фишинговый домен в автозаполнении | Высокая | Высокий | eTLD+1 строгое сравнение |
| Accessibility API утечка | Средняя | Критический | Нативный оверлей для ввода |
| Cold boot attack | Очень низкая | Критический | mlock (частично) |
| Принуждение (rubber hose) | Очень низкая | Критический | Duress password (v2) |

## Принятые риски (не устраняем в v1)

1. **Руткит/буткит уровня ядра** — выходит за TCB, нерешаемо на уровне приложения
2. **Cold boot attack** — mlock снижает риск, полная защита требует TPM+HSM
3. **Квантовые компьютеры** — горизонт 10-15 лет, готовим архитектуру к миграции
4. **Rubber hose cryptanalysis** — duress password запланирован в v2
5. **GPU memory (LeftoverLocals)** — нативный оверлей снижает риск для мастер-пароля

## Угрозы бэкап-системы

### T-20: Компрометация файла бэкапа
**Вектор**: Атакующий получает .vbk файл (через утечку облака, кражу флешки)  
**Контрмера**: Argon2id 4GB — один GPU не может эффективно брутить.
При 24-словной BIP-39 мнемонике пространство ключей = 2048^24. Невозможно.

### T-21: Потеря мнемоники
**Вектор**: Пользователь теряет 24 слова  
**Контрмера**: UX flow с подтверждением записи. Рекомендации 3-2-1 хранения.
VaultPass не может помочь с восстановлением — это намеренное свойство
(нет backdoor).

### T-22: Компрометация OS Keychain
**Вектор**: Вредоносный процесс с правами пользователя читает Keychain  
**Контрмера**: В этом случае атакующий уже имеет полный доступ к системе.
Keychain защищает от физической кражи устройства (без разблокировки),
но не от компрометированной ОС. Это известное ограничение модели.
Для параноиков: отключить quick unlock в настройках.

### T-23: Атака на авто-блокировку
**Вектор**: Атакующий замораживает таймер через отладчик  
**Контрмера**: PR_SET_DUMPABLE=0 + anti-debug проверки в Rust.
При обнаружении отладчика — немедленная блокировка.
