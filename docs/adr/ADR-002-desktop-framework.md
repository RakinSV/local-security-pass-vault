# ADR-002: Desktop фреймворк (Windows + Linux)

**Статус:** Принято  
**Дата:** 2026-06  
**Контекст:** desktop/

## Решение

**Tauri 2** с Rust бэкендом и React UI в WebView.

## Альтернативы

| Критерий | Tauri 2 | Electron | Qt | Flutter Desktop |
|----------|---------|----------|-----|----------------|
| Размер бинаря | ~8 МБ | ~150 МБ | ~30 МБ | ~20 МБ |
| Язык бэкенда | Rust | Node.js | C++ | Dart/C++ |
| Поверхность атаки | Минимальная | Большая (Node.js) | Средняя | Средняя |
| Крипто-интеграция | Нативная (Rust) | Через addon | Нативная | FFI |
| SetProcessMitigationPolicy | Через windows-rs | Через addon | Нативно | Сложно |
| Время разработки UI | Быстрое (Web) | Быстрое (Web) | Медленное | Среднее |
| Win + Linux из одной кодовой базы | ✅ | ✅ | ✅ | ✅ |

## Обоснование

Rust-бэкенд Tauri напрямую вызывает libsodium без IPC overhead.
Все крипто-операции и работа с файлами — в Rust, не в JS sandbox.
WebView только для UI — если WebView скомпрометирован, крипто-операции
недоступны (allowlist IPC команд).

## Последствия

- Нужно знать Rust для бэкенда и TypeScript/React для UI
- WebView2 на Windows обновляется независимо от приложения (мониторить CVE)
- Tauri IPC allowlist нужно настраивать явно (хорошо для безопасности)
